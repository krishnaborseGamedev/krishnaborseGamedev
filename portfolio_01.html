01 — About
From debugging to shipping

I'm a BCA student and self-taught Unity developer. My first real project, CubeBallShooter, started as a basic shooter and turned into a full obstacle-course game — vaulting, climbing, a custom third-person camera, and an animation system built almost entirely by isolating bugs one layer at a time.

I care more about why a system works than just getting it to run. Most of what I learned came from chasing down real bugs: execution-order issues, floating-point thresholds, a missing EventSystem silently breaking every UI button.

I've also worked on Employee Attrition Prediction, a machine learning project that predicts whether an employee is likely to leave a company. It uses Pandas and NumPy for data handling and numerical operations, a Random Forest Classifier built with Scikit-learn for the prediction model, and Streamlit for a simple interactive interface.

I'm currently looking for opportunities to grow as a gameplay programmer on a real development team.

03 — Field Notes
Bugs that taught me the engine
FIXED
RunningJump silently never triggered The locomotion update reset IsRunning to false the instant the jump state began — before the trigger logic could read it. Fixed by caching the value one frame earlier, before any state-blocking logic could overwrite it.
FIXED
Speed parameter never reached the Run threshold Normalized speed asymptotically approached but never hit exactly 1.0 due to floating-point precision. Solved by setting the Blend Tree's Run threshold to 0.9 instead of 1.0.
FIXED
CharacterController.velocity read as zero while walking Two separate Move() calls per frame meant .velocity only reflected the last one (gravity). Combining horizontal and vertical movement into a single Move() call fixed it instantly.
FIXED
Restart button stopped responding to clicks Traced to an accidentally deleted EventSystem GameObject — Unity UI can't process any click without one, and it has no visual presence in the scene to remind you it's missing.
